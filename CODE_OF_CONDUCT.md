# Code Of Conduct

This project welcomes constructive discussion about SPECA, Codex App support,
audit workflows, and reproducible security research.

## Expected Behavior

- Be respectful and assume good faith.
- Keep bug reports and review comments focused on reproducible behavior.
- Avoid posting secrets, private target details, local absolute paths, raw
  worker traces, or uncoordinated third-party vulnerability details.
- Use authorized targets only. Do not use this project to request or coordinate
  unauthorized probing, exploitation, authentication bypass, or service
  disruption.

## Unacceptable Behavior

- Harassment, personal attacks, or discriminatory language.
- Publishing private credentials, tokens, account identifiers, or private local
  environment details.
- Posting unresolved third-party vulnerabilities publicly instead of using the
  target's coordinated disclosure process.
- Using issues or pull requests to request help attacking systems without clear
  authorization.

## Reporting Problems

For conduct problems in public issues or pull requests, contact the maintainers
through GitHub by opening a focused issue or by using repository moderation
tools where available.

For security-sensitive reports, use `SECURITY.md` and GitHub private
vulnerability reporting instead of a public issue.
